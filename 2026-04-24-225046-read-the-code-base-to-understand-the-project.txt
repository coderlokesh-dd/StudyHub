● Here's what the project is, based on exploring the code:

  Student Organizer — Project Overview

  A full-stack React + Express web app that bundles ~10 student productivity
  tools into one dashboard.

  Stack

  - Frontend: React 18 + Vite, react-router, framer-motion, recharts, Supabase
  JS (for auth + file vault)
  - Backend: Node.js + Express 5, PostgreSQL via pg, Multer for uploads
  - Deploy: Vercel/Netlify for the frontend (SPA rewrites); backend runs
  standalone on port 5000
  - Auth: Supabase Auth (email/password) — profiles table + RLS policies in
  supabase-setup.sql

  Features (by page)

  Architecture notes

  - Dual persistence layer: most data goes through the local Express + Postgres
  backend via frontend/src/utils/api.js (13 tables: tasks, notes, subtasks,
  journal, flashcards, exams, timetable_entries, study_sessions, study_log,
  user_data, etc.). Study Vault (semesters/subjects/materials) uses Supabase
  directly with RLS. Auth is Supabase-only.
  - Contexts (frontend/src/contexts/): AuthContext (Supabase session),
  AppContext (all CRUD resources), StudyTimerContext (active session, Pomodoro
  phases, fullscreen), ThemeContext (persisted to localStorage).
  - Floating timer: global pill component that survives route changes so a study
   session keeps running while you browse other pages.
  - Route structure: App.jsx wraps protected pages in Layout (Sidebar +
  BottomNav + ProtectedRoute); Landing is the only public route.

  Recent work (from implementation_plan.md + git log)

  The last five features added were Pomodoro, Flashcards, Exam Countdown, Task
  Subtasks, and Timetable. Current branch is clean on main.

  Gotchas I noticed

  - backend/database.sqlite exists on disk but the backend uses Postgres (pg
  client with DATABASE_URL). The sqlite file looks like legacy.
  - supabase-setup.sql defines vault_* tables with RLS — those are used by Study
   Vault; the other features hit the Express API, which has no per-user
  filtering in its queries. If multi-user support is a goal, that's a gap.

  Let me know what you'd like to dig into or change.
