● Fix applied. Now when the app loads:

  - If lastStudyDate is today → current stays unchanged.
  - If lastStudyDate is yesterday → current stays unchanged (streak still alive
  — log a session today to bump it).
  - If lastStudyDate is older than yesterday → current resets to 0 and persists.
   longest is preserved.

  The increment logic in logStudyTimeFn was already correct (resets to 1 on
  first study after a gap), so the only gap was the dashboard showing a stale
  number until the next session. Now it reflects reality immediately on load.

  One thing to flag: dates use UTC (new Date().toISOString().split('T')[0])
  throughout the app, including studyLog, getWeeklyStudyData, and the streak. If
   you study near local midnight in a non-UTC timezone, days can shift — but
  that's a pre-existing pattern across the codebase, not new to this fix.
